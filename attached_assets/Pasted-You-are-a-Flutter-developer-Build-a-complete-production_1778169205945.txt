TAB 1: SCANNER SCREEN (Home Screen)
═══════════════════════════════════════════════
When the app opens, the user sees a camera icon and "Tap to scan a page" or "Add photos (max 10)". They can take photos or select from gallery. Up to 10 photos allowed.

After photos are added, OCR (google_mlkit_text_recognition) extracts all text from each photo. The extracted text is stored in a variable called "currentPageContext".

The user then sees a text field at the bottom where they can type a question or prompt (e.g., "Explain this page in simple Bengali" or "Solve all math problems on this page" or "Generate a quiz from this content").

The prompt and the page context are sent to the Groq API along with the selected AI Response Language and the system prompt that forces the AI to:
- Give direct, step-by-step explanations.
- Use the selected language (casual everyday form for Indian languages, with scientific terms in English).
- If math or physics problems are present, solve them completely with all steps shown.
- Use plain text only (no LaTeX, fractions as (a)/(b), sqrt as sqrt()).
- If the user didn't ask for a solve, just summarize/explain.

The AI response is displayed beautifully using flutter_markdown. Below the response, there are action buttons:
- "Read Aloud" (speaker icon) – uses flutter_tts to speak the response in the selected Voice Language.
- "Add to Note" – saves the response into the Notebook (with a subject/topic label).
- "Copy" – copies the text to clipboard.

The user can continue asking follow-up questions in the same screen; the context is kept for the session.

AI Response Language picker (not the same as Voice Language) is shown as a dropdown at the top of this screen, allowing the user to choose in which language the AI writes its answer. Default: English.

Voice Language picker is separate (in Settings), controlling the language used by TTS.
